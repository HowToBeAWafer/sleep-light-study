# Sleep Light Study

一个用于睡前光照与视觉注意力研究的网页实验。参与者观看五分钟纯色刺激，并在黑色十字出现时点击屏幕或按键；网页会记录刺激出现、参与者响应和实验环境事件。

## 在线版本

- 推荐入口：[GitHub Pages](https://howtobeawafer.github.io/sleep-light-study/)
- 备用入口：[OpenAI Sites](https://sleep-light-study.dkm26355.chatgpt.site/)

部分网络可能会被 `chatgpt.site` 的 Cloudflare 安全规则拦截，因此参与者应优先使用 GitHub Pages。

## 实验流程

1. 输入参与者编号（Participant ID）。
2. 选择研究者分配的光照条件。
3. 阅读英文说明页，了解注意力任务和控制方式。
4. 确认说明后，网页请求进入全屏并显示三秒倒计时。
5. 参与者观看五分钟纯色刺激；左下角的小倒计时显示剩余有效实验时间。
6. 黑色十字出现时，参与者点击/触摸屏幕，或按 `Space` / `Enter`。
7. 实验中按 `P` 暂停，再按一次 `P` 继续；如需提前终止，依次输入 `E`、`N`、`D`。
8. 实验结束后下载 CSV 或 JSON 数据。

说明页会用英文提醒参与者全程保持专注、等待黑色十字，并在十字出现时按 `Space` 或点击屏幕。暂停期间刺激和注意力任务停止，五分钟倒计时冻结，因此暂停时间不计入有效实验时长。

### 内置测试账号

Participant ID 输入 `test`（不区分大小写）会进入可重复使用的 Test mode：

- 可以不限次数运行完整实验
- 不写入 `localStorage`
- 不进入历史会话或“导出全部会话”
- 结果页只显示试运行统计，不提供 CSV/JSON 下载
- 页面刷新后不会保留该次试运行记录

## 刺激条件

| 条件 | 数字颜色值 | 说明 |
| --- | --- | --- |
| Bright Red | `#FF0000` / RGB `255, 0, 0` | 高数字强度红色 |
| Dim Red | `#660000` / RGB `102, 0, 0` | 低数字强度红色 |
| Bright Blue | `#0000FF` / RGB `0, 0, 255` | 高数字强度蓝色 |
| Dim Blue | `#000066` / RGB `0, 0, 102` | 低数字强度蓝色 |

重要：这些值只控制浏览器像素，不等于物理亮度或照度。正式实验应固定设备、浏览器和显示设置，关闭自动亮度、True Tone、Night Shift 等自动调节，并使用仪器校准屏幕。

## 注意力任务参数

- 倒计时：3 秒，不计入正式实验时长
- 实验时长：`300,000 ms`（5 分钟有效实验时间；暂停时间不计入）
- 计划试次：每次会话固定 `20` 次
- 十字显示时长：`1,800 ms`
- 出现位置：在屏幕安全区域内随机生成
- 出现节奏：开始实验时预先生成固定计划，不受参与者反应快慢影响
- 十字样式：黑色主体配细浅色轮廓，以保证暗红和暗蓝条件下仍可辨认
- 剩余时间：刺激页左下角显示小型倒计时，暂停时保持不变
- 暂停控制：按 `P` 暂停或继续；每次暂停都会记录开始、结束和持续时间
- 提前终止：在运行或暂停状态下依次输入 `E`、`N`、`D`

计划出现时间和真实绘制时间会分别保存。真实出现时间在界面提交后、浏览器下一次绘制前记录，用于计算反应时。

20 次试次分布在 290 秒的可用区间内，每个时间窗宽 14.5 秒。首个计划 onset 约为第 `8.625–15.875` 秒，最后一个约为第 `284.125–291.375` 秒。

## 数据记录

每次会话包含：

- 会话 ID、参与者编号和光照条件
- 实验开始、计划结束和实际结束时间
- 计划有效时长、实际有效时长、墙钟总时长、暂停总时长和十字显示时长
- 是否成功进入全屏
- 十字计划出现时间与实际出现时间
- 点击时间、反应时和输入方式
- 十字坐标和点击坐标
- 命中（`hit`）、漏答（`missed`）、未能按时呈现（`omitted`）和提前终止时正在进行的取消试次（`cancelled`）
- 无十字时的点击以及命中后的多余点击（`false_click`）
- 每次暂停的开始时间、结束时间和暂停时长
- 会话是正常完成（`completed`）还是由 `END` 提前终止（`terminated`）
- 页面隐藏/恢复、进入/退出全屏等环境事件

CSV 始终包含一行 `session_summary`，因此即使参与者在第一个十字出现前就使用 `END`，参与者编号、条件、时间和终止状态也不会丢失。其余事件按照实验经过时间排序；暂停会以独立事件行导出，包含暂停开始、结束和持续时间，误点会以 `false_click` 事件导出。参与者编号会进行电子表格公式注入防护。

使用 `END` 提前终止时，尚未开始的计划仍保留在 JSON 的 `trialPlan` 中，但不会被误记为漏答或 `omitted`；当时正在显示的十字会标记为 `cancelled`。

## 数据保存与隐私

- 当前版本不上传参与者数据。
- 数据保存在参与者当前浏览器的 `localStorage` 中。
- 完成实验后可以下载 CSV 或完整 JSON。
- 开始页可以导出同一设备上保存的全部会话。
- 浏览器存储键为 `sleep-light-study:sessions:v1`。
- 清除浏览器网站数据会删除本地副本。
- 内置 `test` 参与者永远不会写入或导出记录。

如果需要研究者集中收集数据，应增加经过隐私与伦理审查的后端数据库、知情同意流程和访问控制。

## 当前限制

- 浏览器无法保证不同设备上的物理光强一致。
- 当前数据只保存在参与者设备上，不会自动汇总给研究者。
- 浏览器计时与 `requestAnimationFrame` 仍属于近似行为计时；后台节流、卡顿、锁屏或退出全屏可能影响试次，这些情况会尽量记录并标记。
- 黑色十字的知觉对比仍应在正式使用的设备上进行预实验。
- 条件由操作员选择，没有内置随机分组、条件平衡或盲法。
- 刷新或关闭页面后不能续接正在进行的会话，应在结束后立即下载数据。
- Participant ID 只进行去空格和 CSV 安全处理，研究者仍需制定去身份化编号规则。
- 准确率计算为 `hit / (hit + missed)`；`omitted` 和误点不进入准确率。
- 本工具记录的是五分钟视觉注意反应，不测量睡眠结局，也不支持临床效果结论。
