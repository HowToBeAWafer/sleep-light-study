# Sleep Light Study — 搭建日志

本文记录网站从需求定义、实验实现到公开部署的主要过程。日期均为 `2026-07-10`（Asia/Shanghai）。

## 1. 初始需求

目标是搭建一个睡前光照研究网页，包含：

- 约五分钟的纯色视觉刺激
- 亮红、暗红、亮蓝、暗蓝四种条件
- 实验过程中随机出现黑色十字
- 参与者点击屏幕或按键证明持续注意
- 记录十字出现时间和参与者响应时间

## 2. 界面与研究流程

采用深色沉浸式研究界面作为开始页，实验刺激页保持全屏纯色，避免无关界面元素影响刺激。

最终流程：

1. 输入 Participant ID
2. 选择指定条件
3. 阅读英文任务说明并确认开始
4. 三秒倒计时
5. 五分钟有效实验时间的全屏刺激与注意力任务
6. 结果统计与 CSV/JSON 导出

英文说明会在实验开始前明确告知参与者保持专注、等待黑色十字，并在十字出现时按 `Space` 或点击屏幕；同时说明 `P` 暂停/继续和依次输入 `E`、`N`、`D` 提前终止的控制方式。

## 3. 实验引擎实现

初版使用连续随机延迟安排十字。代码审查发现，如果下一次十字从参与者响应后才开始计时，反应较快的人会得到更多试次，影响条件可比性。

随后改为：

- 每次会话开始时预先生成固定 `20` 次试次
- 将 290 秒可用区间划分为 20 个 14.5 秒时间窗，在每个窗口的 25%–75% 区间随机选择计划出现时间
- 所有计划与参与者的点击速度独立
- 十字坐标也在会话开始时预先生成并写入计划

## 4. 计时精度改进

初版在 React 更新界面前记录出现时间，可能让记录时间早于真实绘制时间。

改进后：

- 先同步提交十字界面
- 再在浏览器下一次 `requestAnimationFrame` 中记录视觉 onset
- 保存计划 onset 和实际 onset
- 反应时使用实际 onset 计算
- 持久化操作延后到绘制之后，减少对显示时机的影响

## 5. 注意力与环境记录

除命中和漏答外，增加了以下记录：

- 无目标时的误点
- 页面隐藏和恢复
- 进入和退出全屏
- 全屏请求失败
- 指针坐标、十字坐标和输入方式

如果浏览器隐藏或计时器被节流，已经错过显示窗口的试次不会连续补播，而会标记为 `omitted`。正常完成时会补全所有未能呈现的计划试次，确保完整会话保留 20 次结果；使用 `END` 提前终止时，未来试次只保留在 `trialPlan`，不会被误算为漏答或 `omitted`，正在进行的试次会标记为 `cancelled`。

随后根据实验控制需求增加：

- 在运行时按 `P` 暂停，再按 `P` 继续
- 暂停期间停止刺激和注意力试次，并冻结五分钟有效实验倒计时
- 记录每次暂停的开始时间、结束时间和持续时间
- 在运行或暂停时依次输入 `E`、`N`、`D` 提前终止，并将会话状态记录为 `terminated`
- 正常完成的会话状态记录为 `completed`
- 刺激页左下角显示小型剩余时间倒计时
- 无十字时的点击，以及一次命中后的额外点击，统一记录为 `false_click`

## 6. 刺激可见性处理

审查发现，纯黑十字在 `#660000`、`#0000FF` 和尤其 `#000066` 上的视觉对比较低。

当前处理是保留黑色十字主体，同时增加细浅色轮廓。这样可以提高暗色条件下的可见性，但正式研究仍需要在实际设备上进行对比度与物理亮度校准。

## 7. 数据结构与导出

实现了：

- 会话级 JSON 记录
- 事件级 CSV 导出
- 本地浏览器自动保存
- 同一设备历史会话导出
- CSV 事件按时间排序
- CSV 始终写入 `session_summary`，确保开始后立即 `END` 也保留完整会话元数据
- 参与者编号的电子表格公式注入防护
- 实验结束后的命中数、漏答数、平均反应时和准确率统计
- 暂停事件的开始、结束和持续时间记录
- 实际有效实验时长、墙钟总时长和暂停总时长分别记录
- 正常完成与提前终止的会话状态记录
- `false_click` 事件记录，包括无十字点击和命中后的多余点击

平均反应时只使用 `hit` 试次；准确率为 `hit / (hit + missed)`，不包含 `omitted` 与误点。

暂停不会减少预设的五分钟有效实验时间；继续后，计划试次和剩余倒计时从暂停点恢复。CSV 使用独立的暂停事件行保存暂停开始、结束和时长，JSON 保留完整会话级暂停记录。

当前数据不会传输到服务器。

## 12. 研究解释限制

- 浏览器无法记录实际 lux、光谱功率分布、环境光或显示器校准状态。
- 当前版本不记录设备型号、浏览器版本、刷新率或窗口尺寸。
- 条件由操作员手选，没有内置随机分组、条件平衡、盲法或重复 Participant ID 检查。
- 刷新、关闭、休眠后不会续跑正在进行的实验。
- `localStorage` 可能被用户清理或在隐私模式下受限，完成后必须及时下载。
- 本任务只记录短时视觉注意反应，不记录入睡时间、睡眠质量或其他睡眠结局。

## 13. 内置 Test mode

根据试用需求，加入保留 Participant ID `test`（不区分大小写）：

- 可反复运行完整五分钟实验
- 所有计时、十字和结果统计仍正常工作
- 暂停/继续、`END` 提前终止和左下角倒计时仍正常工作
- `persist()` 在检测到 test participant 后直接返回内存记录，不写入 `localStorage`
- 历史会话导出会过滤 test participant
- 结果页隐藏 CSV/JSON 下载，并明确说明记录未保存
- 进入页面时会清理旧版本可能留下的 test participant 本地记录
- Participant ID 输入框不公开提示 `test`，避免参与者得知研究者测试入口
