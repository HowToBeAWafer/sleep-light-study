# Sleep Light Study — 搭建日志

本文记录网站从需求定义、实验实现到公开部署的主要过程。日期均为 `2026-07-10`（Asia/Shanghai）。

## 1. 初始需求

目标是搭建一个睡前光照研究网页，包含：

- 约五分钟的纯色视觉刺激
- 亮红、暗红、亮蓝、暗蓝四种条件
- 实验过程中随机出现黑色十字
- 参与者点击屏幕或按键证明持续注意
- 记录十字出现时间和参与者响应时间

## 2. 界面与研究流程

采用深色沉浸式研究界面作为开始页，实验刺激页保持全屏纯色，避免无关界面元素影响刺激。

最终流程：

1. 输入 Participant ID
2. 选择指定条件
3. 三秒倒计时
4. 五分钟全屏刺激与注意力任务
5. 结果统计与 CSV/JSON 导出

## 3. 实验引擎实现

初版使用连续随机延迟安排十字。代码审查发现，如果下一次十字从参与者响应后才开始计时，反应较快的人会得到更多试次，影响条件可比性。

随后改为：

- 每次会话开始时预先生成固定 `20` 次试次
- 将五分钟划分为时间窗口，在每个窗口内随机选择计划出现时间
- 所有计划与参与者的点击速度独立
- 十字坐标也在会话开始时预先生成并写入计划

## 4. 计时精度改进

初版在 React 更新界面前记录出现时间，可能让记录时间早于真实绘制时间。

改进后：

- 先同步提交十字界面
- 再在浏览器下一次 `requestAnimationFrame` 中记录视觉 onset
- 保存计划 onset 和实际 onset
- 反应时使用实际 onset 计算
- 持久化操作延后到绘制之后，减少对显示时机的影响

## 5. 注意力与环境记录

除命中和漏答外，增加了以下记录：

- 无目标时的误点
- 页面隐藏和恢复
- 进入和退出全屏
- 全屏请求失败
- 指针坐标、十字坐标和输入方式

如果浏览器隐藏或计时器被节流，已经错过显示窗口的试次不会连续补播，而会标记为 `omitted`。实验结束时也会补全所有未能呈现的计划试次，确保每个会话保留完整的 20 次计划。

## 6. 刺激可见性处理

审查发现，纯黑十字在 `#660000`、`#0000FF` 和尤其 `#000066` 上的视觉对比较低。

当前处理是保留黑色十字主体，同时增加细浅色轮廓。这样可以提高暗色条件下的可见性，但正式研究仍需要在实际设备上进行对比度与物理亮度校准。

## 7. 数据结构与导出

实现了：

- 会话级 JSON 记录
- 事件级 CSV 导出
- 本地浏览器自动保存
- 同一设备历史会话导出
- CSV 事件按时间排序
- 参与者编号的电子表格公式注入防护
- 实验结束后的命中数、漏答数、平均反应时和准确率统计

当前数据不会传输到服务器。

## 8. 质量审查与验证

完成的检查包括：

- TypeScript/React 生产构建通过
- 固定 20 次计划不受响应速度影响
- 十字实际绘制 onset 在界面提交后记录
- CSV 各类事件列对齐
- 页面隐藏导致的过期试次不会被快速补播
- 只接受主指针输入，避免多点触摸污染
- GitHub Pages 页面加载无控制台错误
- Participant ID 输入和条件单选控件已实际验证

## 9. 部署历史

### OpenAI Sites

首次部署地址：

<https://sleep-light-study.dkm26355.chatgpt.site/>

网站随后调整为公开访问，但部分网络仍被站点前的 Cloudflare 安全层返回 HTTP 403。该问题发生在网站代码执行之前，不能通过修改页面访问权限解决。

### GitHub Pages

为避免 `chatgpt.site` 的网络拦截，安装并授权 GitHub 连接，创建公开仓库：

<https://github.com/HowToBeAWafer/sleep-light-study>

随后：

1. 生成可独立运行的静态页面和客户端资源
2. 将资源上传到 GitHub 仓库
3. 添加 GitHub Pages Actions 工作流
4. 在仓库设置中启用 GitHub Actions 作为 Pages 来源
5. 重新执行发布并确认成功
6. 在浏览器中验证页面与交互控件

最终推荐地址：

<https://howtobeawafer.github.io/sleep-light-study/>
